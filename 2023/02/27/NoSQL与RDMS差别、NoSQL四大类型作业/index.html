NoSQL与RDMS差别、NoSQL四大类型作业

NoSQL有什么优点

  • 支持超大规模的数据存储
  • 灵活的可扩展性
  • 灵活的数据模型
  • 可以很好的支持web2.0应用
  • 具有非常好的可用性，能在短时间内迅速返回所需结果

列举至少5个NoSQL与RDMS差别

  • 数据原理方面
    • 关系数据库：具有完备的关系代数理论作为基础
    • NoSQL：缺乏理论基础
  • 数据规模方面
    • 关系数据库：很难实现横向扩展，纵向扩展有限
    • NoSQL：具有非常好的水平可扩展性（集群、分片的添加与删除），对用户透明
      • 很容易通过添加更多节点来支持更大规模的数据
  • 数据库模式方面
    • 关系数据库：要定义严格的数据库模式，而且要严格遵守实现定义的数据库模式
    • NoSQL：数据模型非常灵活
  • 查询效率方面
    • 关系数据库：适当数量级查询效率高，数据量级增大查询效率下降
    • NoSQL：未构建面向复杂查询的索引，查询效率差
  • 事物一致性方面
    • 关系数据库：遵循ACID事务模型可以保证事物强一致性
    • NoSQL：不保证事务的一致性
  • 数据完整性方面
    • 关系数据库：具有保证完整性的完备机制
      • 实体完整性，域完整性，参照完整性，用户定义完整性
    • NoSQL：不能实现完整性约束
  • 可扩展性方面
    • 关系数据库：扩展性一般比较差
    • NoSQL：水平扩展性非常好

NoSQL的四大类型是什么

  • 键值数据库
  • 列族数据库
  • 文档数据库
  • 图数据库

简要说明键值数据库的优缺点及典型应用

  • 优点
    • 扩展性好，灵活性好，吞吐量非常高
  • 缺点
    • 无法存储结构化信息，值和值之间无法反映关联关系
    • 条件查询效率较低，可以key->value查，不能value->key查

典型应用：涉及频繁读写，拥有简单数据模型的应用内容缓存，比如会话、配置文件、参数等存储配置和用户数据信息的移动应用

简要说明文档数据库的优缺点及典型应用

  • 优点
    • 性能好（高并发），灵活性高
    • 提供嵌入式文档功能，将经常查询的数据存储在同一个文档中
    • 既可以根据键来构建索引，也可以根据内容构建索引
  • 缺点
    • 查询性能不太高（相对于关系型数据库）
    • 缺乏同一的查询语法

典型应用：存储、索引并管理面向文档的数据或类似半结构化数据

比如，用于后台具有大量读写操作的网站、使用JSON数据结构的应用、使用嵌套结构等非规范化数据的应用程序
