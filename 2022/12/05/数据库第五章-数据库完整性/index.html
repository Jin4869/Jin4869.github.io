CH5 数据库完整性

数据库完整性

数据库完整性指数据的正确性、相容性和有效性

为维护数据库的完整性，数据库管理系统需实现如下功能：

  • 提供定义完整性约束条件的机制
  • 提供完整性检查方法
  • 进行违约处理

实体完整性

定义实体完整性

在 CREATE TABLE 中用PRIMARY KEY定义

实体完整性检查

  • 检查主码值是否唯一，不唯一则拒绝插入或修改
  • 检查主码的各个属性是否为空，有一个为空就拒绝插入或修改

参照完整性

增删改都可能破坏参照完整性

定义参照完整性

在 CREATE TABLE 中用FOREIGN KEY短语定义哪些列为外码，用REFERENCES短语指明这些外码参照哪些表的主码

参照完整性检查和违约处理
